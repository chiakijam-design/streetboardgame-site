# LIVE 不正利用・荒らし対策

## 初期版の公開条件

- YouTuberは招待制・運営による手動審査制とする。
- 運営は本人性、チャンネルとの関係、連絡先、企画内容、画像利用許諾を確認してから、運営コンソールで招待コードを発行する。
- 招待コードはチャンネルに固定し、発行から90日で失効する。平文は発行時だけ表示し、D1にはSHA-256ハッシュだけを保存する。
- スタッフ用URLと招待コードは別経路で共有する。漏えい時は運営コンソールでURL再発行と招待失効を行う。
- 初期版の応援メッセージは公開しない。将来実装時も、モデレーションと通報・非表示・監査手順が揃うまで公開表示を有効化しない。

## 実装済みの境界

| リスク | 初期版の対策 |
|---|---|
| 他人のチャンネルを使う | 問題生成・企画作成を、審査済みチャンネルに固定した招待コードで認可 |
| 不適切画像 | JPEG/PNG/WebPのみ、10MB以下、画像デコード確認、非公開R2保存、運営承認までは既定画像、却下時はR2から削除 |
| 差別的・攻撃的な参加者名 | NFKC正規化、制御・不可視文字除去、重大な攻撃語・URL・連絡先らしい文字列をサーバー側で拒否 |
| 応援メッセージの誹謗中傷 | 公開フラグをコード上で無効化。公開メッセージ機能は提供しない |
| カード不正利用 | Stripe署名検証、失敗通知、Early Fraud Warning・Dispute・高リスク判定で購入権限を停止して手動審査 |
| 大量参加・大量再接続 | ルーム別IPの参加API制限、ルーム定員、参加予約の冪等化、同一参加者のWebSocketを最大3本に制限 |
| スタッフURL漏えい | URLのホストトークンに加えて端末内の招待コードを要求。URL・招待の個別失効、管理者2FA |

## 手動審査チェックリスト

1. 申請者がチャンネル所有者または正式な運営スタッフであることを、YouTube OAuth、概要欄コード、または本人への既知連絡先で確認する。
2. Stripe本人確認済み名義とチャンネル運営者の関係を確認する。有料販売は所有確認完了まで許可しない。
3. 登録画像が本人・権利者の許諾済みで、露骨な性的表現、暴力、差別、第三者の個人情報を含まないことを確認する。
4. 利用規約、収益分配規約、肖像権ガイドラインへの合意記録を確認する。
5. 招待コードは平文で記録せず、安全な連絡手段で対象スタッフに一度だけ渡す。

## インシデント対応

1. 対象ルームを運営コンソールから強制終了する。
2. スタッフURL・本人URLを再発行し、作成者招待を失効する。
3. 不適切画像を却下して削除し、Stripe不正疑いは購入権限と分配を保留する。
4. 監視イベント、StripeイベントID、ルームコード、発生時刻だけを記録する。荒らし名や応援文の原文を通知・ログへ転載しない。
5. 影響範囲を確認し、必要に応じて障害告知、本人・購入者・関係機関への連絡を行う。

## 本番前の外部設定

- Cloudflare WAF/Rate Limitingで`/api/live/games/*/join`と`/api/live/youtube-candidates`を保護する。
- 悪用が観測されたら、問題生成・企画作成・決済開始にTurnstileを追加する。視聴者投票の毎回認証には使わない。
- Stripe Radarルール、3Dセキュア、Early Fraud Warning、Dispute Webhookを有効化する。
- 本番招待バイパス用の`LIVE_CREATOR_INVITE_BYPASS_TOKEN`は設定しない。これはローカルE2E専用である。
