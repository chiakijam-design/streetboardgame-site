# LIVE安全運用

## 初期上限

- 視聴者上限: 1ルーム50人
- 51人目以降: `POST /api/live/games/:code/join` が `409 participant-limit-reached` を返す
- クライアントの状態確認間隔: 3秒

通常ポーリングでは過去問の個別回答を取得せず、現在問題の選択肢別集計だけをD1で取得する。参加者本人の回答は現在問の本人分だけ、最終結果では本人の全回答だけを一度取得する。

現構成はCloudflare D1を共有し、各端末が定期的に状態を取得する方式。D1の実負荷試験と本番メトリクスを確認するまでは、`LIVE_VIEWER_LIMIT`を引き上げない。

## 予約

- YouTube LIVEの新規保存には未来の予約日時が必須
- 既存予約の開始時刻から前後20時間以内に入る開始時刻は予約不可
- 画面での空き確認後も、企画保存時にD1へ条件付きINSERTを行い競合を再確認
- 予約日時と参加上限はスタッフ用ロビーに再表示
- 開始時にもサーバー側で全体ロックを取得し、別のLIVEが進行中なら開始を拒否
- 最終結果まで完了した時点で全体ロックを解放。異常終了時は最長24時間で自動失効

## 購入用結果画像

購入用画像のプレビューには次の情報を入れる。

- YouTubeチャンネル名
- スタッフが登録したYouTuber画像。未登録時は既定キャラクター
- 予約したライブ配信日
- 視聴者が入力・変更した表示名
- 視聴者本人の正解数

購入前プレビューは540×675px・SAMPLE表示付き。Stripe Checkout / Connect、購入権限、原寸画像の安全な保存先はまだ接続していないため、高画質版のダウンロードボタンは表示しない。

参加者の最終結果では、X投稿・LINE送信・プレビュー画像の保存／共有を利用できる。XとLINEには結果文と`/live`のURLを付け、画像操作は既存シリーズと同じ共通処理を使う。PCではダウンロードし、対応スマホでは画像ファイル付き共有シート、非対応スマホでは本文共有後のダウンロードへフォールバックする。
